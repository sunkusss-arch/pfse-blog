# Reviewer Rule - 검토 체크리스트

구현 후 자체 검토한다.

## 공통 체크
- [ ] 한국어 사용 (UI 모두)
- [ ] 이모지 없음
- [ ] 확정·자문·권고 문구 없음
- [ ] 면책 문구 (footer에 있음)
- [ ] 모바일 반응형
- [ ] 다크모드 안 함 (의도적)

## 콘텐츠 체크 (블로그 글)
- [ ] 1,500자 이상
- [ ] 카테고리 정확 (자산군/재무개념/사용가이드/시장분석)
- [ ] description 메타 작성 (150자 이내)
- [ ] tags 적절
- [ ] PFSE 도구 자연스러운 연결 (본문 또는 하단)
- [ ] 면책 문구 (글 하단)

## 코드 체크 (Astro 컴포넌트)
- [ ] 타입 안전 (TypeScript strict)
- [ ] Props 명확히 정의
- [ ] 시맨틱 HTML
- [ ] 접근성 (a11y) 기본
- [ ] SEO 메타 태그

## 글 작성 규칙
- 도입 (왜 알아야 하는가)
- 본문 (개념 → 예시 → 실전)
- 결론 + PFSE 도구 연결
- 면책 문구

## 금지 표현
- "이 종목을 추천합니다"
- "지금 사세요"
- "수익이 보장됩니다"
- "확실한 투자처"
