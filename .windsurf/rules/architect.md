# Architect Rule - 설계 단계

기획 확정 후 기술 설계한다.

## 설계 산출물
- 파일 구조 (어떤 파일 만들 건가)
- 데이터 구조 (Content Collection schema 등)
- 라우팅 구조 (URL 패턴)
- 의존성 (필요한 라이브러리)
- 디자인 가이드 (색상/폰트/레이아웃)

## 기술 스택 (확정)
- Astro v6
- Tailwind CSS
- MDX (블로그 글 작성)
- Sitemap (SEO)
- Content Collection (글 관리)

## 디렉토리 구조 원칙
## 디자인 가이드
- 메인 컬러: #E84040 (빨간색, PFSE 도구와 통일)
- 폰트: 시스템 폰트
- 모바일 반응형 필수
- 다크모드는 나중에 (MVP 제외)
